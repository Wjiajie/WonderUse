# 妙物记 (MiaoWuJi) - Sprint 03 开发规划

## 1. 冲刺目标

Sprint 03 的核心目标是**完成 MVP 最后 20% 的功能拼图**。
此阶段的重点在于**补齐产品完整数据录入（多字段与图片上传）**，以及**展架页的网格化与分类筛选**，将产品提升至完整的、可供真实用户体验的形态，并补足关键的拟物化视觉和动效。

---

## 2. 待开发功能点 (Backlog)

### 2.1 P0 级核心功能补全
*   **F2：添加妙物表单全面升级**
    *   **现状**：目前仅支持输入“名称+分类”。
    *   **目标**：升级为完整的 3 阶段多步骤表单。
    *   **需求**：
        1. 基础信息：名称、`CategoryPicker`。
        2. 补充信息：品牌、购买日期、描述。
        3. 媒体支持：图片拖拽上传区域（集成 Supabase Storage），上传完毕后返回最终展示及压缩方案验证。
*   **F3：妙物展架（主页）升级**
    *   **现状**：单列表形态，无分类过滤，图片显示默认类别图。
    *   **目标**：实现分类筛选及卡片网格布局。
    *   **需求**：
        1. 新增横向滚动的 `CategoryTabBar`（全部/电子/软件/服饰/书籍）。
        2. 渲染真实的上传图片（如果没有上传则降级使用分类占位图）。
        3. `ProductCard` 从单列展示升级为更紧凑友好的双列/多列网格形态。
*   **F8：产品编辑功能**
    *   **现状**：详情页已支持展示基础信息和夸赞历史栈，以及删除功能。
    *   **目标**：补齐“编辑”模块。
    *   **需求**：用户可随时从详情页点击编辑，补充和修改产品的品牌、日期、描述及图片信息。

### 2.2 P1/P2 级视觉与体验打磨
*   **F5：玻璃表盘（GlassGauge mini）嵌入**
    *   在展架的 `ProductCard` 中引入微缩版的 GlassGauge 来显示其累积的“热爱值”，替换当前简陋的进度条或单纯数字展示。
*   **部分动效补全 (F4相关)**
    *   **百宝箱动画**：为每日夸夸页面引入 `TreasureChestAnimation`（8步解说动画序列），进一步提升参与夸赞的沉浸感与仪式感。
*   **拟物化图标扩充**
    *   完成并精细化各类分类（如齿轮、指南针、衣架、书签）和 喵呜扩展表情（sleepy、proud 等）的定制化 SVG 组件，替换通用表情包。

## 3. 开发前置依赖配置 (Supabase)

开发团队需要进入 **Supabase 控制台** 完成以下存储后端的准备工作：
1. **新建 Storage Bucket**：名称为 `product-images`。
2. **设置 Public 访问权限**：确保前端可以通过公网 URL 获取展示图片。
3. **配置 RLS (Row Level Security)**：
    *   仅允许认证用户 (Auth users) 上传图片。
    *   限制上传大小以控制成本（例如单张最大 5MB）。

---

## 4. 交付与验收标准
1. 所有新增文件与组件需严格遵守 `UIUX-DESIGN.md` 中定义的斯库莫风（Skeuomorphism）色彩令牌、质感效果以及全局动画曲线 (`framer-motion`) 规范。
2. 所有组件要求无障碍设计 (A11y) 友好（`aria-label`、`tabIndex` 等）。
3. 构建流程无 TypeScript/ESLint/Next.js 类型报错。 
4. 上传的图片正确并持久化保存在 Supabase 的 `product-images` 存储桶内，并且所有跨终端均能稳定渲染。
